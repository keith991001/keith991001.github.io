> 出处：Leopold Aschenbrenner《Situational Awareness》(2024) 第一章
> <https://situational-awareness.ai/from-gpt-4-to-agi/>
>
> **核心论点**：只要沿着趋势线数「数量级」（OOM = Order of Magnitude，10 倍 = 1 OOM），到 2027 年出现能承担 AI 研究员/工程师工作的 AGI 是 "strikingly plausible"（惊人地可信）的。

## 0. 全文框架

作者把 GPT-2 → GPT-4 的进步拆成三个可量化的来源，再把同样的方法外推到 2023 → 2027：

1. **算力（Compute）**：训练用的物理计算量增长
2. **算法效率（Algorithmic efficiencies）**：同样效果所需算力的下降，相当于"算力放大器"
3. **去枷锁（Unhobbling）**：RLHF、思维链、工具、长上下文等，把基座模型的潜在能力释放出来

前两项合称**有效算力（effective compute）**，第三项单独算作能力跃迁的额外来源。

## 1. 过去四年：GPT-2 → GPT-4

用"人类学龄"作类比来标定智能水平：

| 模型 | 年份 | 大致水平 | 典型表现 |
|---|---|---|---|
| GPT-2 | 2019 | 学龄前儿童 | 能拼凑半通顺的句子，数数到 5 都不稳 |
| GPT-3 | 2020 | 小学生 | 多段落连贯、简单算术、few-shot 学习，开始有商业价值 |
| GPT-4 | 2023 | 聪明的高中生 | 写可用代码、高中竞赛级数学推理、AP/SAT 超过多数高中生 |

**基准测试被快速"打穿"的证据**：

- **MMLU**（2020 年提出时被当作"终极考卷"）：3 年内被 GPT-4 做到 ~90%
- **MATH**（高中竞赛数学，2021 发布）：当年最好模型只有 ~5%，研究者预测多年内难有突破 → 一年后 50% → 现在 >90%
- **GPQA**（博士级理化生题目）：Claude 3 Opus ~60%，对口领域博士约 80%，眼看也要饱和

**教训**：过去十年反复有人宣判深度学习撞墙（Gary Marcus 对 GPT-2/GPT-3 局限的论断、Bryan Caplan 的经济学期中考试赌局等），全部被下一代模型打脸。"永远不要跟深度学习对赌。"

## 2. Counting the OOMs：数数量级

### 2.1 算力（Compute）

依据 Epoch AI 估算：

- GPT-2 (2019)：~4e21 FLOP
- GPT-3 (2020)：~3e23 FLOP（+~2 OOM）
- GPT-4 (2023)：8e24–4e25 FLOP（再 +1.5~2 OOM）

即 GPT-2 → GPT-4 的原始训练算力增长约 **3,000–10,000 倍**。

- 长期趋势：AI 训练算力约 **0.5 OOM/年**，远快于摩尔定律（每十年才 1–1.5 OOM）——驱动力是**资本投入**（更大集群），不是芯片本身
- 外推 2023–2027：保守 +2 OOM（百亿美元级集群），激进 +3 OOM（千亿美元级集群，如传闻中的 Stargate）

### 2.2 算法效率（Algorithmic efficiencies）

定义：达到同样性能所需训练算力的下降，是"看不见的算力"。

- **ImageNet 时代实证**（2012–2021）：约 0.5 OOM/年，9 年累计 ~4.5 OOM（同性能省 4500 倍算力）
- **LLM 同期趋势**（Epoch AI）：也约 0.5 OOM/年（误差棒更大，因为前沿实验室不公开细节）
- 具体例子：
  - GPT-4 发布时 API 定价 ≈ GPT-3 发布时定价，性能却天差地别 → 粗算约一半提升来自算法
  - Chinchilla scaling laws：一项就 3 倍+（0.5+ OOM）
  - MoE（Mixture of Experts）、AdamW、SwiGLU、位置编码等各类改进
  - MATH 基准的推理成本：不到 2 年降了约 3 OOM（1000 倍）
- GPT-2 → GPT-4 估计算法贡献 **1–2 OOM**；外推 2023–2027 再来 **~2 OOM**（上限 3）

### 2.3 数据墙（The data wall）—— 最大的不确定性

- 前沿模型已经把互联网基本刮完：Llama 3 用了 ~15T token；Common Crawl 去重后约 30T token；公开代码（GitHub）只有低万亿级
- 重复训练收益递减（约 16 个 epoch 之后基本没用）
- **作者的乐观论据**：
  - 人类不是靠"扫读全网"学习的——精读、内心独白、和同伴讨论、做题、纠错。当前 LLM 的学习方式极其低效，**样本效率（sample efficiency）还有巨大改进空间**
  - **AlphaGo 先例**：第一步模仿人类棋谱（≈预训练），第二步自我对弈（self-play）超越人类。LLM 的"第二步"（合成数据 / self-play / RL）正是各实验室重注押注的方向
  - 业内人士（如 Dario Amodei）普遍乐观："有很多不同的办法"
- 结论：数据墙让时间线的方差变大——可能拖慢进度，也可能因高质量合成数据反而加速

### 2.4 Unhobbling：从基座模型到可用系统

核心概念：基座模型潜力巨大，但被默认形态"绑住了手脚"，小改动就能解锁大能力。

已经发生的 unhobbling：

1. **RLHF**：InstructGPT 论文显示，小模型 + RLHF 在人类偏好上 ≈ 100 倍大的未对齐模型——ChatGPT 能商用全靠它
2. **思维链（CoT）**：2022 年才普及，数学/推理上相当于 10 倍+ 有效算力。类比：不许人打草稿直接报答案，当然做不好
3. **Scaffolding（脚手架）**：让多个模型分工（规划→提案→批评）。HumanEval 上带脚手架的 GPT-3.5 能胜过裸 GPT-4；SWE-Bench 上 GPT-4 裸跑 ~2%，Devin 式 agent 脚手架 ~14–23%
4. **工具使用**：浏览器、代码执行——"以前等于不许人用计算器和电脑"
5. **上下文长度**：GPT-3 2k → GPT-4 32k → Gemini 1.5 Pro 1M+。Gemini 靠上下文里塞一本词典语法书就学会了新语言 Kalamang
6. **发布后的 posttraining 改进**：GPT-4 发布后 MATH 50%→72%、GPQA ~40%→~50%、LMSYS Elo +100

量化：Epoch AI 调研显示脚手架/工具等通常等效 **5–30 倍**算力；METR 的 agent 任务上同一基座 GPT-4 从 5%（裸）→ 20%（发布版）→ ~40%（加工具脚手架）。

### 2.5 从聊天机器人到 Agent 同事（chatbot → agent-coworker）

当前模型仍被绑着的地方，以及对应的解锁方向：

1. **入职问题（onboarding problem）**：GPT-4 像"5 分钟前刚入职的聪明新人"，没有公司文档、Slack 历史、代码库背景 → 用超长上下文把模型"入职"
2. **测试时算力悬置（test-time compute overhang）**：现在模型每题只"思考"几百 token（≈人类 5 分钟）。按 ~100 token/分钟换算：
   - +1 OOM（数万 token）≈ 半个工作日
   - +2 OOM（数十万 token）≈ 一个工作周
   - +4 OOM（数百万 token）≈ 数个月的连续工作
   解锁方式：教会模型 System II 式的外层循环——规划、纠错、迭代。旁证：棋类研究（Jones 2021）中约 1.2 OOM 测试时算力可抵 1 OOM 训练算力
3. **使用电脑**：现在的模型是"只能发短信联系的、关在箱子里的人"→ 多模态 + computer use 后可以开 Zoom、查资料、发邮件、用内部工具

**终态：drop-in remote worker（即插即用的远程员工）**——入职一个月背景知识、在 Slack 里协作、独立完成数周项目。作者认为这不需要比 GPT-4 强太多的基座，大头在解除枷锁。经济价值可能呈"音爆式"不连续爆发：中间态需要大量集成苦工，一旦到达 agent 形态则即插即用。

## 3. 未来四年（2023–2027）：账本汇总

| 来源 | GPT-2→GPT-4（4 年） | GPT-4→2027（4 年，预测） |
|---|---|---|
| 算力 | +1.5~2 OOM | +2~3 OOM |
| 算法效率 | +1~2 OOM | +1~3 OOM（最可能 ~2） |
| **有效算力合计** | **+4.5~6 OOM** | **+3~6 OOM（最佳猜测 ~5）** |
| Unhobbling | 聊天机器人化（质变） | Agent 化 / 远程员工化（质变） |

- 直观化："GPT-4 训练花了 3 个月；2027 年的头部实验室能用 1 分钟训出 GPT-4 水平的模型。"
- 类比："AI 的成长速度约是人类儿童的 3 倍。你的 3 倍速小孩刚高中毕业，转眼就要抢你的工作了。"
- 预期终点：超过博士和领域专家的模型 + agent 形态 → **能自动化几乎所有可远程完成的认知工作**
- 作者的 AGI 定义：**能自动化 AI 研究员/工程师工作的系统**。一旦 AI 研究自身被自动化，数百万个自动研究员可能把十年的算法进步压缩到一年内 → "AGI 只是随后而来的超级智能的一小口预尝"

## 4. Addendum: It's this decade or bust（不成功便成仁的十年）

**论点**：对 AGI 的不确定性应该放在"还差几个 OOM"上，而不是"还差几年"上——因为 OOM 的产出速度极不均匀，**本十年是特殊的**：

1. **烧钱的一次性红利**：$1M 集群（当年已惊人）→ $10B → $100B → 或许 $1T 封顶。再往后受 GDP 约束，只能跟着经济 ~2%/年 增长
2. **硬件的专用化红利趋于耗尽**：CPU → GPU → Transformer 专用芯片。榨干之后回落到摩尔定律速度
3. **算法的低垂果实**：现在聪明人和资金正大规模涌入，容易的改进被快速摘完；之后放缓

推论：2020s 约产出 ~10 OOM，之后每十年可能只有 1–2 OOM。所以——

- 如果 2027 前后没实现，**2030 年前实现不了的话，之后可能要等很久**
- 作者的众数预测：**AGI 年份落在本十年内的某一年**

## 5. 批判性阅读备忘

- 全文最强的部分是**历史趋势的实证**（算力、算法效率两条 0.5 OOM/年 的线），最弱的部分是**unhobbling 能否兑现 agent 化**——这一步没有趋势线可外推，属于定性判断
- "数据墙可解"主要依赖业内人士的乐观表态和 AlphaGo 类比，无公开证据（各实验室结果保密）
- 站在 2026 年回看这份 2024 年的预测：test-time compute overhang 的判断已被推理模型（o1/o3、扩展思考）部分验证；agent 化（computer use、长任务）也在发生——这条时间线目前没有明显跑偏
- OOM 记账法的隐含假设：**能力随有效算力平滑、可预测地增长**（scaling laws 在 15 个 OOM 上成立）。若涌现能力不连续，外推的误差会很大
